### 使用说明

#### 前提
1，修改config文件中的本地路径信息
2，修改config文件中的邮箱信息
3，启动邮件解析功能测试，确保邮件可以正确读取
----
4，每天自动执行之前，先人工打开浏览器，登录一下投保网站

#### 每日操作说明
1，执行邮件解析流程，检查本地生成的文件（**本地文件是核心，用于控制整个流程**）
  核实以下信息：
  - 投保类型：快递，非快递；决定去哪里投保
  - 货物类型：决定投保时的选择，是否普货
  - 投保日期（不能为空），是否正确
  - 投保状态：未投保，已投保，已下载保单，已发送邮件（**4个流程的控制状态**）
